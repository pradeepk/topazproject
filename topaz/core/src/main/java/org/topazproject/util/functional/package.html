A set of utilities to make functional programming easier.

Functional programming requires closures, which will not be generally available until JDK 1.7. This package contains templates and methods which provide similar functionality to closures via "functors". Since functor syntax is quite verbose, the class and method names are intentionally terse, though hopefully with sensible mnemonics.

Functions are represented by classes starting with the name "Fn". An optional "E" may follow, indicating that the function may throw an exception. Finally, a number at the end of the classname indicates that the number of arguments for the function.

The interface templates are:

  • Fn - Function taking no arguments and will not throw an exception.
  • FnE - Function taking no arguments and may throw an exception.
  • Fn1 - Function taking 1 argument and will not throw an exception.
  • Fn1E - Function taking 1 argument and may throw an exception.
  • Fn2 - Function taking 2 arguments and will not throw an exception.
  • Fn2E - Function taking 2 arguments and may throw an exception.

The C class represents methods for interacting with Collections. These include methods for traditional head/tail operations common in functional programs, and functional methods, such as map and foreach.

The F class contains static methods for currying. It will curry N argument functors to N-1 argument functors.
